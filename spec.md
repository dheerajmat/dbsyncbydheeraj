<table><tr><td><p><strong>DB SYNC &amp; SCHEMA</strong></p><p><strong>COMPARISON TOOL</strong></p><p>Node.js SaaS Application</p><p>Product Specification  |  Architecture Guide  |  Core Logic Reference</p></td></tr></table><h1><strong>Project Overview</strong></h1><p>This document describes the design, architecture, and implementation details for a Node.js SaaS-style tool that compares and synchronizes two PostgreSQL databases. It covers both Prisma-based and direct PostgreSQL comparison modes, schema diff output, SQL sync generation, and a minimal frontend.</p><h2><strong>Core Feature Summary</strong></h2><table><tr><td><p><strong>Feature</strong></p></td><td><p><strong>Description</strong></p></td><td><p><strong>Mode</strong></p></td></tr><tr><td><p>DB Connection Input</p></td><td><p>Accepts credentials for DB A and DB B</p></td><td><p>Both</p></td></tr><tr><td><p>Prisma Comparison</p></td><td><p>Uses prisma db pull to introspect and diff schemas</p></td><td><p>Prisma</p></td></tr><tr><td><p>Direct PG Comparison</p></td><td><p>Uses pg + information_schema queries</p></td><td><p>Direct</p></td></tr><tr><td><p>Diff Output</p></td><td><p>Tables, columns, types, constraints diff view</p></td><td><p>Both</p></td></tr><tr><td><p>SQL Sync Generator</p></td><td><p>Produces CREATE TABLE / ALTER TABLE SQL</p></td><td><p>Both</p></td></tr><tr><td><p>Sync Execution</p></td><td><p>Optional: run generated SQL on target DB</p></td><td><p>Both</p></td></tr><tr><td><p>REST API Layer</p></td><td><p>Express endpoints for all operations</p></td><td><p>Both</p></td></tr><tr><td><p>SaaS UI</p></td><td><p>Form inputs, tabs, diff viewer, SQL preview</p></td><td><p>Both</p></td></tr></table><h1><strong>1. Database Connection Input</strong></h1><p>Each connection form accepts the following five fields for both Database A and Database B:</p><table><tr><td><p><strong>Field</strong></p></td><td><p><strong>Description</strong></p></td></tr><tr><td><p>Host</p></td><td><p>Hostname or IP address of the PostgreSQL server</p></td></tr><tr><td><p>Port</p></td><td><p>Default: 5432</p></td></tr><tr><td><p>Username</p></td><td><p>Database user with read permissions</p></td></tr><tr><td><p>Password</p></td><td><p>Database user password</p></td></tr><tr><td><p>Database Name</p></td><td><p>Target database to connect to</p></td></tr></table><h1><strong>2. Comparison Modes</strong></h1><h2><strong>Mode 1: Prisma-Based Comparison</strong></h2><p>This mode uses the Prisma CLI to introspect both databases and produces a structured schema comparison based on the generated .prisma files.</p><ul><li>Run prisma db pull against both databases using their respective connection URLs</li><li>Parse the resulting .prisma schema files into an AST (Abstract Syntax Tree)</li><li>Compare models (tables), fields (columns), types, and relations</li><li>Surface differences in a structured diff output</li></ul><p>Introspection command:</p><table><tr><td><p>npx prisma db pull --url="postgresql://user:pass@host:5432/dbname"</p></td></tr></table><h2><strong>Mode 2: Direct PostgreSQL Comparison (No Prisma)</strong></h2><p>This mode uses the pg library and queries the PostgreSQL system catalog and information_schema views directly. No Prisma installation is required.</p><ul><li>Connect to both databases using the pg client</li><li>Query information_schema.tables for table names</li><li>Query information_schema.columns for column names and types</li><li>Query pg_constraint for constraint definitions</li><li>Compare results programmatically via the diff engine</li></ul><p>Schema fetch query:</p><table><tr><td><p>const getSchema = async (client) =&gt; {</p><p>  const tables = await client.query(`</p><p>    SELECT table_name</p><p>    FROM information_schema.tables</p><p>    WHERE table_schema = 'public'</p><p>  `);</p><p>  const columns = await client.query(`</p><p>    SELECT table_name, column_name, data_type</p><p>    FROM information_schema.columns</p><p>    WHERE table_schema = 'public'</p><p>  `);</p><p>  return { tables: tables.rows, columns: columns.rows };</p><p>};</p></td></tr></table><h1><strong>3. Difference Output</strong></h1><p>Comparison results are organized into four categories:</p><table><tr><td><p><strong>Category</strong></p></td><td><p><strong>Description</strong></p></td></tr><tr><td><p>missingTablesInB</p></td><td><p>Tables that exist in A but not in B</p></td></tr><tr><td><p>missingTablesInA</p></td><td><p>Tables that exist in B but not in A</p></td></tr><tr><td><p>columnDifferences</p></td><td><p>Columns missing in B or A for matching tables</p></td></tr><tr><td><p>typeMismatches</p></td><td><p>Same column exists but with different data types</p></td></tr></table><p>Diff engine implementation:</p><table><tr><td><p>const compareSchemas = (schemaA, schemaB) =&gt; {</p><p>  const diff = {</p><p>    missingTablesInB: [],</p><p>    missingTablesInA: [],</p><p>    columnDifferences: []</p><p>  };</p><p>  const tablesA = schemaA.tables.map(t =&gt; t.table_name);</p><p>  const tablesB = schemaB.tables.map(t =&gt; t.table_name);</p><p>  diff.missingTablesInB = tablesA.filter(t =&gt; !tablesB.includes(t));</p><p>  diff.missingTablesInA = tablesB.filter(t =&gt; !tablesA.includes(t));</p><p>  schemaA.columns.forEach(colA =&gt; {</p><p>    const match = schemaB.columns.find(</p><p>      colB =&gt; colB.table_name === colA.table_name &amp;&amp;</p><p>              colB.column_name === colA.column_name</p><p>    );</p><p>    if (!match) {</p><p>      diff.columnDifferences.push({</p><p>        type: 'missing_column_in_B',</p><p>        table: colA.table_name,</p><p>        column: colA.column_name</p><p>      });</p><p>    } else if (match.data_type !== colA.data_type) {</p><p>      diff.columnDifferences.push({</p><p>        type: 'type_mismatch',</p><p>        table: colA.table_name,</p><p>        column: colA.column_name,</p><p>        A: colA.data_type,</p><p>        B: match.data_type</p><p>      });</p><p>    }</p><p>  });</p><p>  return diff;</p><p>};</p></td></tr></table><h1><strong>4. SQL Sync Generator</strong></h1><p>The SQL generator converts diff output into executable SQL statements. Sync direction can be configured as A to B or B to A.</p><h2><strong>Generated Statement Types</strong></h2><ul><li>CREATE TABLE — for tables missing in the target database</li><li>ALTER TABLE ADD COLUMN — for columns missing in the target table</li><li>ALTER TABLE MODIFY COLUMN — for columns with type mismatches</li></ul><h2><strong>SQL Generator Implementation</strong></h2><table><tr><td><p>const generateSQL = (diff) =&gt; {</p><p>  const queries = [];</p><p>  diff.missingTablesInB.forEach(table =&gt; {</p><p>    queries.push(`-- Need CREATE TABLE ${table}`);</p><p>  });</p><p>  diff.columnDifferences.forEach(col =&gt; {</p><p>    if (col.type === 'missing_column_in_B') {</p><p>      queries.push(</p><p>        `ALTER TABLE ${col.table} ADD COLUMN ${col.column} TEXT;`</p><p>      );</p><p>    }</p><p>    if (col.type === 'type_mismatch') {</p><p>      queries.push(</p><p>        `-- Type mismatch: ${col.table}.${col.column} (A: ${col.A}, B: ${col.B})`</p><p>      );</p><p>    }</p><p>  });</p><p>  return queries;</p><p>};</p></td></tr></table><table><tr><td><p><strong>Optional: Sync Execution</strong></p><p>A dedicated Execute button can run the generated SQL directly on the target database via a /execute API endpoint. This should include a confirmation dialog and dry-run preview before execution.</p></td></tr></table><h1><strong>5.
